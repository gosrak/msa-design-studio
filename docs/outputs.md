# 산출물 가이드

⬇ 산출물 출력 메뉴에서 각 산출물을 XLSX 또는 PPTX 로 다운로드. 통합 출력은 7종 PPTX + 통합 XLSX 일괄 다운로드.

---

## ① 이벤트 도출 (Domain Events)

**용도**: 도메인 이벤트 인벤토리 — 비즈니스 분석 단계 결과물

### XLSX 시트 — 이벤트도출
| NO | 시스템 | 도메인 | 서브도메인 | 상세기능명 | 이벤트 | 커맨드 |
|---:|--------|--------|------------|------------|--------|--------|

### PPTX
도메인별 그리드 (5×4 = 20 이벤트/페이지). 오렌지 포스트잇 + 검은 시퀀스 번호.
- 첫 슬라이드: 타이틀 + 총 이벤트 수
- 도메인 N개 × 페이지 N장

---

## ② 이벤트 요소 도출 (Event Storming Elements)

**용도**: 워크숍 결과 풀 매트릭스 — 도메인 전문가 검토용

### XLSX 시트 — 이벤트요소도출
17 컬럼 풀 매트릭스 (시스템·도메인·서브도메인·상세기능명·이벤트·커맨드·액터·애그리거트·읽기/정책·외부시스템·핫스팟·데이터저장소·Redis 사용·메시징·CQRS·유스케이스·인바운드어댑터)

### PPTX
**한 페이지 = 한 서브도메인의 이벤트 클러스터** (4×2 그리드, 한 카드 240×320 비례).

각 카드는 이벤트 스토밍 보드의 7요소 포스트잇 비례를 그대로 PPT 좌표로 환산:

```
        [HOTSPOT]                         [EXTERNAL]
     (회전 -15°, 핑크)                   (우상단)
[CMD]      [EVENT]
(좌상)     (중앙, 오렌지)
                                       
[ACTOR]                            [AGGREGATE]
(좌하)                             (우하, 스택)
       [POLICY  ────  bottom band]
```

> 워크숍 화면을 그대로 인쇄할 수 있는 형식.

---

## ③ 액터 도출 (Actors)

**용도**: 사용자/시스템 권한·역할 명세 (RBAC 기초)

### XLSX 시트 — 액터도출
| NO | 액터명 | 유형 | 분류 | 액터설명 | 주요역할 | 비고 | 연관 이벤트 수 | 시스템(중복제거) | 대표 도메인 |

### PPTX
3×3 = 9 액터/페이지. 노랑 포스트잇 + 우상단 유형 칩 (사람/내부/외부)

각 액터 카드:
- 액터명 (큰 글자)
- 유형 칩 (우상단)
- 분류 / 주요역할 / 설명
- 비고 + 👤 연관 이벤트 N건 (하단 라인)

---

## ④ 외부시스템 정의 (External Systems)

**용도**: 외부 의존성 명세 — Anti-Corruption Layer 설계 기초

### XLSX 시트 — 외부시스템정의
| NO | 시스템명 | 포트명 | 유형 | 프로토콜 | 인증방식 | 연계방향 | 운영주체 | SLA | 설명 | 비고 | 연관 이벤트 수 | 연계 도메인 | 연관 액터 |

### PPTX
2×3 = 6 외부 시스템/페이지. 핑크 포스트잇.

메타 칩 (우측):
- 유형 (REST/Kafka/MQ/...)
- 프로토콜 (HTTPS/AMQP/...)
- 인증 (OAuth2/API Key/mTLS)
- 연계방향 (Inbound/Outbound/Both)
- SLA (Critical/High/...)

→ 헥사고날 아키텍처의 **아웃바운드 어댑터** 설계 직결.

---

## ⑤ 데이터저장소 정의 (Data Stores)

**용도**: 폴리글랏 퍼시스턴스 매핑

### XLSX 시트 — 데이터저장소정의
| NO | 저장소명 | Repository명 | 유형 | 제품 | 일관성 | 아키패턴 | 용도 | 보존정책 | 설명 | 비고 | 연관 이벤트 수 |

### PPTX
2×3 카드/페이지. **카테고리별 색상**:
- RDB (연두) · NoSQL (연파랑) · Cache (앰버) · Search (연빨강)
- TimeSeries (연보라) · Message (연분홍) · File (연회색)
- Object (연자주) · Graph (연초록) · VectorDB (크림)

각 카드:
- 저장소명 (큰 글자)
- 메타 칩 (유형·제품·일관성·아키패턴)
- 용도 / 보존정책 / 설명
- 🗄 연관 이벤트 N건

---

## ⑥ 바운디드 컨텍스트 (Bounded Contexts)

**용도**: 마이크로서비스 후보 도출

### XLSX 시트 — 바운디드컨텍스트
| 시스템 | 도메인 | 서브도메인(MS후보) | 마이크로서비스명 | 이벤트수 | 정책 | 외부연계 | 핫스팟 | 데이터저장소 | 메시징 | CQRS |

### PPTX
3 카드/행 그리드. 도메인별 슬라이드.

각 카드 = 마이크로서비스 후보:
- MS 명 (자동 생성: `{서브도메인}서비스`)
- 서브도메인 (한국어)
- 메타 pill: 이벤트 N · 외부 연계 · 정책 · 핫스팟
- 기술 행:
  - 🟨 DB (저장소 목록)
  - 🟨 MSG (메시징 토픽)
  - 🟨 CQRS (Command/Event/Read 분포)

---

## ⑦ 헥사고날 아키텍처 (Hexagonal / BFD)

**용도**: 마이크로서비스 청사진 — 개발팀 인계 문서

### XLSX 시트 — 헥사고날아키텍처
| 시스템 | 도메인 | 서브도메인 | MS명 | 이벤트수 | 애그리거트(목록) | 정책수 | 핫스팟수 | 인바운드 어댑터 | 유스케이스(목록) | 커맨드(목록) | 액터(목록) | Repository(저장소) | 외부 포트(시스템) | 메시징/이벤트 |

### PPTX — **BC 1개 = 3 슬라이드**

#### Slide A · BFD 도출 1/2
좌측 6 카드 그리드 + 우측 Key Findings 1~6:
1. 서비스·응용기능 분할 및 정의
2. 권한체계 정의
3. 기존 서비스 분석 (AS-IS)
4. 주요 데이터 분석
5. 데이터·어플리케이션 상관관계
6. 도메인 간 데이터 결합도

#### Slide B · BFD 도출 2/2
좌측 6 카드 + 우측 Key Findings 7~12:
7. 도메인 내 결합도 분석
8. 도메인 도출
9. 마이크로서비스 식별
10. MSA 애플리케이션 아키텍처 정의
11. MSA 애플리케이션 아키텍처 상세
12. 마이크로서비스 명세

#### Slide C · 헥사고날 다이어그램
풀 헥사 다이어그램 (16:9):
- 3중 nested 헥사곤 (Infrastructure / Application / Domain)
- Driving / Driven 측면 색상 띠
- CQRS 칩 (C=N · E=N · R=N)
- MS Set 좌·우 (자기 도메인 / 다른 도메인)
- 메시지브로커 좌(Inbound) / 우(Outbound)
- 액터 패널
- 좌·우 어댑터 4개
- 4 I/F 박스 (Inbound · Outbound × 3)
- Command / Event 6슬롯
- 16종 화살표 (방향성)
- OR Mapper + DB
- 유스케이스 스트립

---

## ★ 통합 XLSX

7시트를 한 권에 통합. 컬럼명·시트명 표준화로 후속 분석/리포트 자동화에 적합.

| 형식 | 단일 종류 | 통합 |
|------|-----------|------|
| XLSX | 각 종 단일 시트 | ✅ 7시트 통합 한 권 |
| PPTX | 각 종 단일 PPT | — (개별 다운로드) |

UI 의 ⬇ 산출물 출력 드롭다운에서 단건 또는 ⏬ 통합 버튼으로 선택.
