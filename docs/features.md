# 기능 상세 (Features)

## 🧩 1. 이벤트 스토밍 캔버스

DDD 기반 디지털 이벤트 스토밍 보드. 알베르토 브란돌리니의 표준 색상·표준 절차를 그대로 따름.

### 17 필드 이벤트 카드 스키마

| 필드 | 설명 | 다중값 | 예시 |
|---|---|:---:|---|
| system | 상위 시스템명 | | 온라인쇼핑몰 |
| domain | BC(도메인)명 | | 주문 |
| subdomain | 서브도메인 (기능 묶음) | | 주문 |
| detail | 상세기능명 | | 카드결제 주문 생성 |
| event | 도메인 이벤트 (과거형) | | 주문생성됨 |
| command | 커맨드 (현재형 동사) | | 주문생성 |
| actor | 액터 | ✓ | 고객, 시스템 |
| aggregate | 애그리거트 (영문 PascalCase) | ✓ | Order, OrderLine |
| readPolicy | 읽기/정책 (자유 텍스트) | | 재고/가격 검증 + Saga 시작 |
| external | 외부시스템 | ✓ | PG시스템, 카카오페이 |
| hotspot | 핫스팟 (이슈/리스크) | | 결제실패, 재고부족 |
| dataStore | 데이터저장소 | ✓ | OrderDB, InventoryDB |
| redis | Redis 사용 | | 재고락, 카트(7일 TTL) |
| messaging | 메시징 토픽 | | order.created (Kafka) |
| cqrs | CQRS 분류 | | Command/Event/Read |
| useCase | ApplicationService 메서드명 | | PlaceOrder |
| inboundAdapter | 인바운드 어댑터 | | REST API/Kafka 구독/Scheduler/Webhook |

### 마스터 데이터 (3종)

이벤트와 별개로 액터/외부시스템/데이터저장소를 풍부한 메타데이터로 별도 관리:

| 마스터 | 메타 컬럼 |
|---|---|
| **액터** | 액터명·유형(사람/내부/외부)·분류·설명·주요역할·비고 |
| **외부시스템** | 시스템명·포트명·유형·프로토콜·인증·연계방향·운영주체·SLA·설명 |
| **데이터저장소** | 저장소명·Repository명·유형·제품·일관성·아키패턴·용도·보존정책 |

> 헥사고날 아키텍처에서 **포트명·Repository명**은 자동 추출되어 PPT 헥사 다이어그램에 반영됩니다.

## 📊 2. 7가지 산출물 자동 생성

### XLSX (8 시트)

| 시트 | 내용 |
|---|---|
| **이벤트도출** | 시스템·도메인·서브도메인·이벤트·커맨드 (간단 형식) |
| **이벤트요소도출** | 17 필드 풀 매트릭스 |
| **액터도출** | 액터 마스터 + 연관 이벤트 수·시스템·도메인 통계 |
| **외부시스템정의** | 외부 마스터 + 연관 이벤트·도메인·액터 |
| **데이터저장소정의** | 저장소 마스터 + 연관 이벤트 |
| **바운디드컨텍스트** | 서브도메인 → MS 후보 매핑, 이벤트수·정책·핫스팟·DB·메시징 |
| **헥사고날아키텍처** | BC 단위 풀 헥사 정의 (애그리거트·인바운드/아웃바운드 포트) |
| **전체통합** | 위 7시트 통합 한 권 |

### PPTX (16:9 와이드)

| PPT | 슬라이드 구성 |
|---|---|
| **이벤트 도출** | 도메인별 오렌지 포스트잇 5×4 그리드 |
| **이벤트 요소 도출** | 한 페이지 한 클러스터 (이벤트+커맨드+핫스팟+외부+액터+애그+정책 7요소 포스트잇 비례 배치) |
| **액터 도출** | 액터 카드 3×3 그리드, 유형 칩, 연관 이벤트 카운트 |
| **외부시스템 정의** | 메타 칩 (유형·프로토콜·인증·SLA), 운영주체·설명 |
| **데이터저장소 정의** | 카테고리별 색상 카드 (RDB/NoSQL/Cache 등 10종) |
| **바운디드 컨텍스트** | MS 후보 카드 + DB/MSG/CQRS 메타 |
| **헥사고날 아키텍처** | **BC 1개 = 3 슬라이드 (BFD-A · BFD-B · 헥사 다이어그램)** |

### 헥사고날 슬라이드 상세

> BC 1개 → 3 슬라이드 패턴 (4.2.5 BFD 도출 문서화 표준)

**Slide A** — 6 카드 그리드 (서비스 분할·권한·기존 분석·데이터·CRUD·결합도) + Key Findings 1~6

**Slide B** — 6 카드 그리드 (도메인내 결합·도메인 도출·MS 식별·MSA 정의·MSA 상세·MS 명세) + Key Findings 7~12

**Slide C** — 헥사고날 다이어그램:
- 3중 nested 헥사곤 (외부영역=Infrastructure / 내부영역=Application / 서비스도메인=Domain)
- Driving / Driven 측면 색상 띠 (좌측 연파랑, 우측 연주황)
- CQRS 칩 (Command/Event/Read 카운트)
- MS Set 좌·우 (자기 도메인 / 다른 도메인)
- 메시지브로커 좌(Inbound) / 우(Outbound)
- 액터 패널 (4명 + …외 N명)
- 좌측 어댑터 2개 (메시지 컨슈머 / API 퍼블리싱)
- 우측 어댑터 2개 (API 호출 프락시 / 이벤트 메시지 처리)
- 4 I/F 박스 (Service Inbound / Proxy Outbound / Event Outbound / Repository Outbound)
- Command 6슬롯 + Event 6슬롯 + 16종 화살표
- OR Mapper + DB 정보
- ApplicationService 유스케이스 스트립 (6 칩 + 오버플로우)
