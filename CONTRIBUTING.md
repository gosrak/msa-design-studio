# Contributing

이 저장소는 **소개·문서·샘플 전용** 입니다. 실제 소스는 비공개로 관리되므로 코드 PR 은 받지 않습니다.

## 환영하는 기여

### 📝 문서 개선
- 오타·표현 다듬기
- 누락된 패턴/사례 추가 (`docs/`)
- 다국어 번역 (영어 등)

### 📊 샘플 추가
다른 도메인 사례를 `samples/` 에 추가:
- 양식지에 도메인 작성 → XLSX 첨부
- HTML export → 같이 첨부
- README 에 도메인 요약 표 추가

도메인 후보:
- 헬스케어 (병원·환자·보험)
- 은행코어 (계좌이체·대출·외환)
- 물류·창고 (입고·출고·재고)
- 구독서비스 (가입·갱신·해지)
- 미디어·콘텐츠 (업로드·변환·CDN)
- IoT (센서·임계치·알림)
- 인사·HR (채용·근태·평가)

### 🐛 버그 신고
[버그 신고 템플릿](.github/ISSUE_TEMPLATE/bug_report.yml) 으로 데모 사이트 이슈 알려주세요.

### ✨ 기능 요청
[기능 요청 템플릿](.github/ISSUE_TEMPLATE/feature_request.yml) 으로 아이디어를 알려주세요.

### 💬 도메인 컨설팅 / 사용 문의
[질문 템플릿](.github/ISSUE_TEMPLATE/question.yml) 으로 등록.

---

## PR 절차 (문서·샘플)

1. Fork
2. 브랜치 생성 — `docs/...` 또는 `samples/...`
3. 변경 후 PR 생성
4. PR 제목: `[Docs] 헥사고날 슬라이드 설명 보강` 등 명확하게
5. 가능하면 한국어 + 영문 병기

---

## 행동 강령 (Code of Conduct)

상대방을 존중하고, 도메인 지식·경험 차이를 이해하며, 건설적 토론을 지향합니다.
